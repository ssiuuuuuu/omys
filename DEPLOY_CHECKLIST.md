# OMYS 공개 전 체크리스트

## 필수 설정

- [ ] `PLACES_PROVIDER=google`로 변경하고 결제·쿼터가 설정된 Google Places API 키를 입력한다.
- [ ] Google Cloud에서 Places API (New)를 활성화하고 API 키에 API/서버 IP 제한을 적용한다.
- [ ] `ADMIN_API_KEY`, PostgreSQL 비밀번호를 각각 긴 무작위 값으로 교체한다.
- [ ] `TMAP_API_KEY`를 서버 비밀 환경변수로 설정하고 보행자 길찾기 권한·쿼터를 확인한다.
- [ ] `CORS_ORIGINS`, `FRONTEND_URL`, `VITE_API_BASE_URL`을 실제 HTTPS 도메인으로 설정한다.
- [ ] `.env`가 Git과 배포 이미지에 포함되지 않았는지 확인한다.

## 인프라와 데이터

- [ ] 운영 PostgreSQL에 `alembic upgrade head`를 실행한다.
- [ ] DB 자동 백업, 보존 기간, 복구 테스트를 설정한다.
- [ ] HTTPS 인증서와 HTTP→HTTPS 리디렉션을 적용한다.
- [ ] API/프런트 헬스체크와 오류 알림을 설정한다.
- [ ] 외부 API 쿼터·비용 알림과 타임아웃 모니터링을 설정한다.

## 모바일 실기기 QA

- [ ] iOS Safari와 Android Chrome에서 방 생성→초대→추첨→공개를 각각 검증한다.
- [ ] 서로 다른 일반/시크릿 브라우저로 2명 이상 동시 참여를 검증한다.
- [ ] 위치 허용·거부·낮은 정확도에서 공개 흐름을 확인한다.
- [ ] Web Share API, 링크 복사, 카카오톡/인스타그램 붙여넣기 레이아웃을 확인한다.
- [ ] 장소 공식 페이지·전화번호가 공개 후 정상 열리는지 확인한다.
- [ ] 320px, 390px, 430px 화면과 가로 회전에서 주요 버튼이 가려지지 않는지 확인한다.

## 운영 정책

- [ ] 서비스 이용약관·개인정보 처리방침·위치정보 안내 문구를 법률 검토 후 게시한다.
- [ ] mock 장소가 운영 배포에 포함되지 않도록 환경변수를 재확인한다.
- [ ] 잘못된 영업시간/장소 정보 신고 및 고객 문의 채널을 준비한다.
- [ ] 관리자 분석 API가 공개 인터넷에서 직접 노출되지 않게 방화벽 또는 인증 프록시를 둔다.
